Q86 (CI/CD triggers on source code change) — picked [A, C] (S3 with change detection + CodeCommit with commit-driven start). Some sources prefer [A, D] if they include the polling option, but commit-driven is the modern recommended pattern.
Q132 (concert seat selection with two downstream Lambdas) — picked C (SNS fanout to both Lambdas). A (SNS FIFO → SQS FIFO) is also defensible if you read "must process in order" strictly, but the question doesn't really require strict ordering.

[B, D] is defensible too if you read "client-side encryption via bucket policy" as a valid mechanism, but bucket policies don't enforce client-side encryption directly.
Q174 (initializing SDK outside Lambda handler) — B (runtime environment reuse), the canonical Lambda performance pattern.
Q235 (CloudWatch ThrottlingException) — D (exponential backoff retry). A (limit increase) is a longer-term answer but D is the immediate best practice.
Q283 (multimedia app with guest + authenticated users, two answers) — picked [B, D] (Cognito identity pool for unauthenticated + separate roles for each user type). [A, D] is sometimes selected by treating user pools as the entry point, but identity pools are the canonical AWS choice for unauthenticated access.
